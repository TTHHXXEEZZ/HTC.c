# เอกสารส่งมอบระบบ (Handoff Document) - HTC Workplace Connect

เอกสารฉบับนี้จัดทำขึ้นเพื่อใช้สำหรับส่งมอบและสรุปวิธีการติดตั้ง การรันระบบ โครงสร้างซอร์สโค้ด และข้อมูลจำเป็นสำหรับการนำระบบฐานข้อมูลสถานประกอบการและการฝึกงานนี้ไปพัฒนาต่อ

---

## 1. ข้อมูลภาพรวมของระบบ (Project Overview)

**HTC Workplace Connect** เป็นเว็บบอร์ดระบบกลางในการรวบรวมสถานประกอบการและแบ่งปันประสบการณ์การฝึกงานของนักศึกษาวิทยาลัยเทคนิคหาดใหญ่ (HTC)
- **ระบบล็อกอิน:** รองรับการล็อกอินจำลองด้วย Google Account โดยบังคับให้ใช้อีเมลวิทยาลัยที่ลงท้ายด้วย `@htc.ac.th` เท่านั้น
- **การปักหมุดแผนที่:** บูรณาการแผนที่อินเตอร์แอคทีฟ **Leaflet.js (OpenStreetMap)** ทำให้สามารถค้นหาพิกัด ปักหมุด และกดเปิดตำแหน่งบน Google Maps ได้ทันทีโดยไม่ต้องผูกบัตรเครดิตเพื่อขอ API Key
- **ระบบตัวกรอง:** กรองสถานประกอบการตามประเภทธุรกิจ แผนกวิชาหลัก (รองรับครบทั้ง 10 แผนกวิชา) และคะแนนรีวิว
- **การรีวิวและยอดผู้เข้าชม:** นักศึกษาสามารถให้คะแนน 1-5 ดาว เขียนรายละเอียดงาน บอกเบี้ยเลี้ยงรายวัน และแนบรูปภาพขณะทำงานได้ รวมถึงระบบติดตามยอดผู้เข้าชมเว็บไซต์และยอดคนเข้าดูของแต่ละบริษัทแบบเรียลไทม์

## 3. ไลบรารีและเทคโนโลยีที่ใช้ (Tech Stack & Dependencies)

- **Frontend Framework:** React 18+ (ผ่าน Vite)
- **Styling:** Vanilla CSS (ใช้ตัวแปร CSS Variables ควบคุมทั้งระบบเพื่อความลื่นไหลและแก้ไขง่าย)
- **Icons:** `lucide-react` (ไอคอนดีไซน์มินิมอล ทันสมัย)
- **Interactive Maps:** `leaflet` และ `react-leaflet` (แสดงผลแผนที่แบบไม่ต้องใช้คีย์)
- **Data Persistence:** Browser `localStorage` (จัดเก็บข้อมูลรีวิว ยอดวิว และข้อมูลบริษัทในเครื่องเบราว์เซอร์ของผู้ใช้ทันที)

## 4. ขั้นตอนการเตรียมระบบและรันในเครื่อง (Setup & Installation)

ในการทดสอบระบบหรือรันเพื่อสาธิตในเครื่องคอมพิวเตอร์ของคุณ ให้ดำเนินการดังนี้:

1.  **เปิด Command Prompt / PowerShell** หรือ Terminal ในโฟลเดอร์ของโปรเจกต์ (`e:\HTC`)
2.  **ติดตั้งไลบรารีที่จำเป็น (Dependencies):**
    ```bash
    npm install
    ```
3.  **รันระบบในโหมดพัฒนา (Development Mode):**
    ```bash
    npm run dev
    ```
4.  เปิดเว็บเบราว์เซอร์ไปที่ลิงก์ที่แสดงบนจอ (โดยปกติคือ [http://localhost:5173/](http://localhost:5173/)) เพื่อทดสอบใช้งาน

## 5. วิธีการนำโค้ดขึ้น GitHub (Putting to GitHub)

เนื่องจากโปรเจกต์ในเครื่องยังไม่ได้ติดตั้ง Git Repository และผูกกับ GitHub ปลายทาง คุณสามารถทำตามขั้นตอนด้านล่างนี้ในโฟลเดอร์ `e:\HTC` เพื่อพุชโค้ดขึ้น GitHub ของคุณเอง:

1.  **สร้าง Git Local Repository:**
    ```bash
    git init
    ```
2.  **เพิ่มไฟล์ทั้งหมดเข้าคิวเตรียมคอมมิต (ไฟล์ขยะจะถูกกรองออกตาม `.gitignore` ที่มีอยู่แล้ว):**
    ```bash
    git add .
    ```
3.  **คอมมิตไฟล์เริ่มต้น:**
    ```bash
    git commit -m "Initial commit: HTC Workplace Connect - Floating Canvas Layout"
    ```
4.  **สร้างที่เก็บข้อมูลบน GitHub (สร้างเป็น Repository ใหม่บนเว็บ GitHub.com) แล้วนำ URL มาเชื่อมต่อ:**
    ```bash
    git remote add origin https://github.com/USERNAME/REPOSITORY_NAME.git
    ```
    *(เปลี่ยน `USERNAME` และ `REPOSITORY_NAME` ให้เป็นของคุณเอง)*
5.  **เปลี่ยนชื่อกิ่งหลักเป็น main และพุชโค้ดขึ้น GitHub:**
    ```bash
    git branch -M main
    git push -u origin main
    ```

## 6. แนวทางการพัฒนาหลังบ้านต่อในอนาคต (Future Backend Integration)

ในเวอร์ชันปัจจุบัน ข้อมูลทั้งหมดถูกเชื่อมต่อผ่าน `src/services/api.js` ซึ่งเรียกใช้งาน `localStorage` ทำให้ข้อมูลจะบันทึกอยู่ในเครื่องของเบราว์เซอร์เครื่องนั้น ๆ เท่านั้น หากต้องการเปลี่ยนระบบให้เป็นแบบออนไลน์ใช้งานร่วมกันได้ สามารถปรับเปลี่ยนระบบหลังบ้านได้โดย:
- **เปลี่ยน API Layer:** แก้ไขฟังก์ชัน `getWorkplaces`, `addWorkplace`, `getReviews`, `addReview` ในไฟล์ `src/services/api.js` ให้เป็นการส่งคำสั่ง `fetch()` ไปยัง Web API จริง
- **ตัวเลือกหลังบ้านแนะนำ:**
  - **Firebase / Supabase (แนะนำสูงสุด):** เป็นบริการ BaaS (Backend-as-a-Service) ที่ให้ฐานข้อมูลแบบ Realtime และระบบสมาชิก โดย Supabase สามารถเชื่อมต่อกับแอป React ได้ง่ายมากและมีฐานข้อมูล PostgreSQL ในตัว
  - **Node.js + Express + MongoDB/PostgreSQL:** เขียน API Server แยกต่างหากเพื่อประมวลผลข้อมูลลงฐานข้อมูลจริง
